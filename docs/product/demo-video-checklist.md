# Demo Video Checklist

## Goal

Record one `3` minute video that proves the current Chinese workspace flow:
model connection, AI 同事 selection, coding plan approval, execution, review, and
verification.

## Before Recording

- Run database migrations.
- Start `api`, `worker`, and `web`.
- Confirm the demo user can log in.
- Confirm one valid DeepSeek model connection exists in settings.
- Confirm the workbench can launch a coding workflow.

## Shot List

### 0:00-0:20

- Show the compact shell and active workspace.
- Show that navigation pages use concise labels.

### 0:20-0:50

- Open `设置 > 模型连接`.
- Validate or show an already-valid DeepSeek connection.
- Confirm no secret is visible after saving.

### 0:50-1:30

- Open `工作台`.
- Choose `编码`.
- Show recommended AI 同事 cards.
- Delete one teammate and confirm the remaining set still works.

### 1:30-2:15

- Start the workflow.
- Show plan pending approval.
- Approve the plan and show execution updates.

### 2:15-2:45

- Open channel detail and switch tabs.
- Confirm the channel title and workspace context remain stable.
- Show recovery actions for a failed activity if present.

### 2:45-3:00

- Summarize what is ready for customer testing.
- State that additional model choices are later expansion, not the current
  setup path.

## Recording Rules

- Prefer one continuous take.
- Keep browser zoom high enough that the sidebar and main panel are legible.
- Avoid implementation jargon in narration.
